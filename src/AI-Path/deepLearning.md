# Learning Path — DeepLearning.AI para Software Developers

## Objetivo del path

Este learning path está pensado para un perfil de **Frontend / Software Developer** que quiere pasar de “usar herramientas de IA” a **diseñar, construir y explicar productos reales con IA**.

La idea no es convertirse de entrada en Machine Learning Researcher, sino desarrollar criterio técnico para integrar IA en productos: LLMs, RAG, agentes, workflows, evaluación, performance y experiencia de usuario.

---

## Resultado esperado

Al terminar este path deberías poder:

- Entender los fundamentos de Machine Learning y Generative AI.
- Explicar cómo funcionan los LLMs a nivel conceptual.
- Diseñar aplicaciones con RAG sobre datos propios.
- Construir agentes que usen herramientas externas.
- Integrar IA en tu workflow de desarrollo.
- Evaluar respuestas de modelos y reducir alucinaciones.
- Pensar en costos, latencia, caching y performance.
- Construir un proyecto de portfolio sólido con IA aplicada.

---

# Etapa 1 — Fundamentos de IA y Machine Learning

## Objetivo

Construir una base conceptual sólida sin quedarse atrapado durante meses en teoría o matemática pesada.

---

**Prioridad:** Media  
**Nivel:** Introductorio  
**Tipo:** Conceptual / estratégico

### Por qué hacerlo

Sirve para entender qué puede y qué no puede hacer la IA, detectar oportunidades de aplicación y ordenar el lenguaje general del área.

### Qué deberías sacar de este curso

- Diferencia entre IA, Machine Learning y Deep Learning.
- Casos reales de aplicación.
- Limitaciones de la IA.
- Cómo pensar oportunidades de producto con IA.

### Link

https://www.deeplearning.ai/courses/ai-for-everyone/

# Etapa 4 — RAG, embeddings y vector databases

## Objetivo

Aprender a construir aplicaciones que responden usando datos propios: documentos, bases de conocimiento, tickets, CVs, documentación técnica o información interna de una empresa.

---

## 7. Retrieval Augmented Generation

**Prioridad:** Muy alta  
**Nivel:** Intermedio  
**Tipo:** Técnico / aplicado

### Por qué hacerlo

RAG es una de las habilidades más importantes para construir productos con IA. Muchas aplicaciones reales no necesitan entrenar modelos desde cero, sino conectar modelos con datos propios de forma confiable.

### Qué deberías sacar de este curso

- Arquitectura de RAG.
- Information retrieval.
- Chunking.
- Embeddings.
- Vector databases.
- Retrieval.
- Generation.
- Evaluación de sistemas RAG.
- Problemas comunes: mala recuperación, contexto irrelevante, hallucinations.

### Link

https://www.deeplearning.ai/courses/retrieval-augmented-generation/

---

## 8. Building Applications with Vector Databases

**Prioridad:** Alta  
**Nivel:** Intermedio  
**Tipo:** Práctico

### Por qué hacerlo

Te ayuda a entender cómo usar bases vectoriales para construir búsqueda semántica, RAG y aplicaciones que trabajan con similitud de contenido.

### Qué deberías sacar de este curso

- Semantic search.
- Embeddings.
- Vector databases.
- Similarity search.
- Aplicaciones prácticas con datos no estructurados.

### Link

https://www.deeplearning.ai/short-courses/building-applications-vector-databases/

---

## 9. Advanced Retrieval for AI

**Prioridad:** Media / Alta  
**Nivel:** Intermedio  
**Tipo:** Optimización

### Por qué hacerlo

Después de aprender RAG básico, este curso te ayuda a mejorar la calidad de recuperación. Esto es clave porque un RAG malo no falla por el modelo: falla porque trae mal contexto.

### Qué deberías sacar de este curso

- Técnicas avanzadas de retrieval.
- Mejoras en ranking.
- Mejoras en relevancia.
- Estrategias para reducir respuestas incorrectas.
- Evaluación de recuperación.

### Link

https://www.deeplearning.ai/short-courses/advanced-retrieval-for-ai/

# Etapa 5 — Agentes y workflows

Aprender a construir sistemas que no solo responden texto, sino que pueden usar herramientas, consultar APIs, tomar decisiones en pasos y ejecutar workflows.

**Nivel:** Intermedio  
**Tipo:** Conceptual / práctico

- Tool use.
- Planning.
- Reflection.

# Etapa 6 — Producción, evaluación y performance

## Objetivo

Aprender a construir aplicaciones con IA que funcionen fuera de una demo: evaluación, testing, costos, caching, latencia e inferencia.

---

## 13. Automated Testing for LLMOps

**Prioridad:** Media / Alta  
**Nivel:** Intermedio  
**Tipo:** Testing / producción

### Por qué hacerlo

Las apps con LLMs son difíciles de testear porque las respuestas no siempre son determinísticas. Este curso apunta a crear pipelines de evaluación para validar cambios.

### Qué deberías sacar de este curso

- Testing para aplicaciones con LLMs.
- Evaluación automática.
- Pipelines de CI.
- Métricas de calidad.
- Validación de outputs.
- Detección de regresiones.

### Link

https://www.deeplearning.ai/short-courses/automated-testing-llmops/

---

## 14. Semantic Caching for AI Agents

**Prioridad:** Media / Alta  
**Nivel:** Intermedio  
**Tipo:** Performance / costos

### Por qué hacerlo

El caching semántico permite reutilizar respuestas no solo cuando el texto coincide exactamente, sino cuando el significado es similar. Esto ayuda a bajar costos y mejorar latencia.

### Qué deberías sacar de este curso

- Semantic caching.
- Reducción de costos.
- Mejora de latencia.
- Reutilización de respuestas.
- Performance en agentes y apps con IA.

### Link

https://www.deeplearning.ai/short-courses/semantic-caching-for-ai-agents/

---

## 15. Fast and Efficient LLM Inference with vLLM

**Prioridad:** Media  
**Nivel:** Intermedio / avanzado  
**Tipo:** Infraestructura / inferencia

### Por qué hacerlo

No lo haría al principio. Es útil cuando ya entendés LLMs, RAG y agentes, y querés profundizar en deployment, serving e inferencia eficiente.

### Qué deberías sacar de este curso

- Serving de LLMs open-source.
- Optimización de inferencia.
- Benchmarking.
- Throughput.
- Latencia.
- Costos de infraestructura.

### Link

https://www.deeplearning.ai/short-courses/fast-efficient-llm-inference-with-vllm/

# Proyecto final recomendado

## AI Career Copilot for Frontend Developers

Una app de portfolio que combine frontend, producto e IA aplicada.

---

## Features sugeridas

- Upload de CV.
- Upload o input de job description.
- RAG sobre experiencias, proyectos y skills.
- Comparación entre CV y job description.
- Generación de respuestas para entrevistas.
- Agente que sugiera mejoras en el CV.
- Respuestas con fuentes o referencias internas.
- Streaming de respuestas.
- Estados claros de loading, error y success.
- Feedback del usuario para evaluar calidad.
- Caching semántico para consultas repetidas.
- Deploy público.

---

## Stack sugerido

### Frontend

- Next.js
- TypeScript
- Tailwind
- Framer Motion
- shadcn/ui o Radix UI

### Backend

- Node.js o Python
- API routes o FastAPI
- OpenAI / Anthropic SDK
- Vector database

### AI

- RAG
- Embeddings
- LLM calls
- Agentic workflow
- Evaluation
- Semantic caching

---

## Qué demostraría este proyecto

- Que sabés construir interfaces modernas.
- Que entendés IA aplicada a producto.
- Que podés trabajar con documentos y datos propios.
- Que sabés diseñar workflows con LLMs.
- Que pensás en UX, performance, costos y confiabilidad.
- Que no solo consumís IA: podés construir software alrededor de IA.

---

# Recomendación final

No intentes hacer todos los cursos como si fueran una checklist infinita. El objetivo no es coleccionar certificados, sino construir criterio y proyectos.

El orden más importante es:

1. Fundamentos.
2. LLMs.
3. Software development con IA.
4. RAG.
5. Agentes.
6. Evaluación y producción.
7. Proyecto final.

Con eso, tu perfil puede evolucionar de:

> Frontend Developer con experiencia en React y Next.js

a:

> Frontend / Product Engineer capaz de construir interfaces modernas e integrar IA en productos reales con criterio técnico.

Ese posicionamiento es mucho más fuerte para los próximos años.
