Lección 0001 · Estructuras de datos

Big-O y el trade del hash map

Ponerle nombre y costo a lo que ya hacés — para defenderlo en voz alta.

Por qué esta lección, y por qué primera. En tu playground/ ya tenés group-by, key-by-id, count-by. Ya usás hash maps, papá — sin nombrarlos. Lo que separa a un senior de un junior en una entrevista de estructuras de datos no es escribir el loop: es decir la complejidad correcta mientras lo escribís. Eso es lo que entrenamos hoy.

1 · Qué mide Big-O (y qué no)

Big-O no mide tiempo en milisegundos. Mide cómo crece el costo cuando crece el input n. Por eso tirás las constantes y te quedás con el término dominante: O(2n + 5) se reporta como O(n). No te importa el +5 cuando n es un millón.

La intuición que necesitás hoy es una sola: un loop adentro de otro loop sobre el mismo input es O(n²). Y la trampa es que ese segundo loop muchas veces está escondido dentro de un método.

La trampa que casi nadie ve. arr.includes(x), arr.find(...) y arr.indexOf(x) son O(n) — recorren el array. Si los llamás adentro de un loop, multiplicás: O(n) × O(n) = O(n²). Se ve inocente. No lo es.

2 · El antes: O(n²)

Problema clásico ("two sum"): dado un array y un target, ¿hay dos números que sumen target? La versión naive:

// Por cada número, busco su complemento en el RESTO del array.
function hasPair(nums, target) {
  for (let i = 0; i < nums.length; i++) {
    for (let j = i + 1; j < nums.length; j++) {   // ← loop adentro de loop
      if (nums[i] + nums[j] === target) return true;
    }
  }
  return false;
}
// O(n²) tiempo · O(1) espacio

3 · El después: O(n) con un Set

El insight: mientras recorro, me pregunto "¿ya vi el complemento que necesito?". Si guardo lo que vi en un Set, esa pregunta es un lookup O(1).

function hasPair(nums, target) {
  const seen = new Set();                // memoria: lo que ya recorrí
  for (const n of nums) {
    if (seen.has(target - n)) return true; // ← lookup O(1), no O(n)
    seen.add(n);
  }
  return false;
}
// O(n) tiempo · O(n) espacio

El trade en una frase: cambiaste tiempo por memoria. Una pasada O(n) para construir el Set, y la búsqueda interna pasó de O(n) a O(1). De O(n²) a O(n). Ese es el patrón que vas a repetir el 80% de las veces.

Profundizá la tabla de costos en la referencia de Big-O — tenela abierta al lado mientras practicás.

4 · Recuperá de memoria (no leas — pensá)

Antes de ver la respuesta, contestá vos: dado const nums = [3, 1, 4, 1, 5], ¿cuál es la complejidad de detectar si hay duplicados usando un Set, y cómo lo escribís? Decilo en voz alta. Después abrí esto:

function hasDup(nums) {
  const seen = new Set();
  for (const n of nums) {
    if (seen.has(n)) return true;   // lookup O(1)
    seen.add(n);
  }
  return false;
}
// O(n) tiempo · O(n) espacio

Si dijiste "O(n) tiempo, O(n) espacio, una pasada con Set" — ganaste la lección. Si dijiste "O(n²)", releé la sección 1: el .has() de un Set es O(1), no O(n). Esa es justo la distinción que te hace senior.
