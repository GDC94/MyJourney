# Mission: Senior Frontend Interview Readiness (~5 años)

## Why
German quiere poder sentarse en cualquier entrevista de Frontend Senior y rendir con
calma en todos los frentes — no por una entrevista puntual, sino para tener un dominio
durable que lo deje siempre listo. El objetivo final: conseguir (y elegir) roles senior
de frontend sin que ningún tipo de pregunta lo agarre desprevenido.

## Success looks like
- Resuelve un ejercicio de live coding narrando su proceso, tipando todo, manejando los 4 estados.
- Reconoce el patrón detrás de un problema de estructuras de datos y **dice su complejidad Big-O en voz alta** correctamente.
- Defiende fundamentos de React/Next hablando: rendering, RSC, hooks por dentro, reconciliación.
- Responde behaviorals con estructura (STAR) y ejemplos reales propios.
- Contesta las preguntas clásicas ("closures", "virtual DOM", "event loop") sin titubear.
- Explica cómo dirige agentes de IA: cuándo, cómo configura `CONTEXT.md` / `DESIGN.md`, y por qué la IA puede equivocarse.

## Constraints
- **Sin fecha fija** → construimos storage strength (espaciado, interleaving), no cram.
- Español rioplatense (voseo). Es la lengua de trabajo.
- Ya tiene base sólida: array methods, data transforms, fetch + React Query (ver LR-0001).
- Sesiones cortas, una victoria tangible por lección. La working memory es chica.

## Out of scope (por ahora)
- DSA de competencia hardcore (DP avanzada, grafos pesados). Frontend rara vez los pide.
- Backend / system design de infra. Foco en frontend.
- Cramear para una entrevista concreta — si aparece una con fecha, se reabre esa decisión.

## Frentes (un mismo "why", varias skills)
1. Estructuras de datos & algoritmos  ← **arrancamos acá**
2. Live coding & velocidad
3. Teoría React / Next
4. Preguntas clásicas de entrevista
5. Behaviorals (STAR)
6. IA / Claude / agentic workflows (CONTEXT.md, DESIGN.md)
