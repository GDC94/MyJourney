# Senior Frontend Interview Resources

> Fuentes de alta confianza. El conocimiento de las lecciones se saca de acá, no de
> memoria. Anotá siempre qué cubre cada una y cuándo recurrir a ella.

## Knowledge

### Estructuras de datos & algoritmos
- [Big-O Cheat Sheet — bigocheatsheet.com](https://www.bigocheatsheet.com/)
  Tabla canónica de complejidades de estructuras y algoritmos comunes. Usar para: recordar el costo de una operación antes de decirlo en voz alta.
- [NeetCode — Practice & Patterns](https://neetcode.io/practice)
  El estándar de facto para patrones de entrevista (Blind 75 / NeetCode 150). Usar para: elegir el próximo patrón a practicar (hash map, two pointers, sliding window, BFS/DFS).
- [Top LeetCode Patterns — DesignGurus](https://www.designgurus.io/blog/top-lc-patterns)
  Resumen de los patrones que más aparecen. Usar para: mapear "este problema huele a ___ patrón".

### Live coding / React Query
- [Status Checks in React Query — TkDodo](https://tkdodo.eu/blog/status-checks-in-react-query)
  El mantenedor de React Query sobre el orden de chequeos de estado y por qué importa. Usar para: defender el orden data → error → loading.
- [TanStack Query Docs (v5)](https://tanstack.com/query/latest)
  Fuente primaria. Usar para: API exacta (`isPending` vs `isLoading` en v5, `placeholderData`, etc.).

## Wisdom (Communities)
- [r/leetcode](https://reddit.com/r/leetcode) — para sanity-check de dificultad y patrones que aparecen.
- [r/reactjs](https://reddit.com/r/reactjs) — debates de fundamentos y novedades de React/Next.

> Nota: el usuario aún no confirmó si quiere sumarse a comunidades. Preguntar antes de empujar.
