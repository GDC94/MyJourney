La regla de oro: Big-O describe cómo CRECE el costo cuando n crece — no el tiempo exacto. Se quedan con el término dominante y se tiran las constantes: O(2n + 5) → O(n). Loops anidados sobre el mismo input multiplican: O(n)·O(n) = O(n²).

El trade que más vas a usar

Doble loop O(n²) → hash map O(n). Si estás buscando algo dentro de un loop (includes, find, indexOf), tenés un O(n) escondido adentro de otro O(n). Pre-cargá un Map o Set de una pasada (O(n)) y reemplazá la búsqueda interna por un lookup O(1). Cambiás tiempo por memoria.

Frase para la entrevista

"La solución naive es O(n²) porque por cada elemento recorro el resto del array. Puedo bajarlo a O(n) tiempo / O(n) espacio usando un Map: una pasada para construirlo, y el lookup pasa a ser O(1)."
